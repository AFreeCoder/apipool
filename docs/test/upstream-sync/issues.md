# 上游同步遗留问题

- [ ] Kiro 账号未进入后台主动 token 刷新候选。
  - 现象：`ListOAuthRefreshCandidatePage` 仅选择 `oauth` 与 `setup-token`，不会选择 `type=kiro`；Kiro 刷新执行器虽已注册，但主动刷新周期无法把 Kiro 账号送达执行器。
  - 归因：在同步前基线 `579166b874c6672f5429c68cf217c8dab998669d` 中已存在相同筛选限制，不是 v0.1.156 同步引入的回归。
  - 本次决定：不混入上游同步发布修复；后续应作为独立 feature 明确 Kiro 主动刷新需求、候选类型模型与回归用例。

- [ ] 异步图片任务尚未使用可恢复的持久任务队列。
  - 现象：任务由当前进程内 goroutine 执行；容器重启或进程退出后不会自动续跑。若退出发生在上游已完成或已计费、Redis 完成态尚未写入之间，调用方可能拿不到结果。
  - 归因：由 v0.1.158 上游新增异步图片任务设计引入，gstack 评审确认。
  - 本次止损：异步图片功能默认关闭；任务超过执行时限仍为 `processing` 时，后续轮询会把它明确标记为 `failed`，不再持续展示 24 小时处理中。部署门禁要求生产 `image_storage.enabled=false`，在持久队列完成前不得启用。
  - 后续方向：另起 feature 设计 Redis Stream/List 持久队列、租约与幂等领取、启动恢复、上游完成但落库失败的对账/补偿，以及相应重启故障测试。

- [ ] 鉴权缓存持久失效事件尚未覆盖完整授权快照。
  - 现象：数据库 outbox 触发器只覆盖安全、状态、IP 与归属等部分字段，但 `APIKeyAuthSnapshot` 还包含配额、余额、并发、组费率、模型路由和媒体权限；服务层提交后的缓存失效错误未向调用方传播。
  - 风险：未来多实例部署或 Redis 故障期间，其他实例可能在 TTL 到期前继续使用旧授权快照。
  - 本次决定：当前生产仍是单实例，进程内失效覆盖主要路径，不在上游同步中扩展 outbox 设计。
  - 后续方向：另起 feature 建立授权快照字段到 outbox 触发器/服务失效路径的完整映射，并增加 Redis 故障、进程退出和跨实例一致性测试。

- [ ] Prompt Audit 缺少提示词数据的保留期限、字段加密与隐私治理。
  - 现象：启用后会把完整提示词和事件内容以 PostgreSQL `TEXT` 明文持久化，未发现自动 TTL、字段级加密或脱敏策略。
  - 风险：提示词可能包含个人信息、客户数据或业务机密，长期明文保留会扩大泄露与合规影响面。
  - 本次决定：功能默认关闭；生产部署门禁要求保持关闭，因此不阻塞本次同步。
  - 后续方向：另起 feature 明确保留期限、删除任务、字段加密/密钥轮换、访问审计和管理员风险提示，完成前不得在生产开启。
